# Prompts de Implementacao

Use estes prompts quase sem adaptacao. Preencha apenas o escopo solicitado.

## Prompt curto para gerar interface

```text
Use $onfly-retrospective-design.
Implemente a interface de [produto] para o perfil [perfil].
Baseie-se no roteiro em references/screen-playbooks.md.
Entregue uma experiencia pronta para demo, com dados mockados realistas de travel tech B2B.
Mantenha a interface responsiva, com narrativa visual forte e componentes reutilizaveis.
Inclua o momento de IA como texto gerado dentro da interface.
Nao crie funcionalidades fora do escopo.
```

## Prompt para Wrapped Onfly do gestor

```text
Use $onfly-retrospective-design.
Implemente o Wrapped Onfly para o perfil gestor.
Crie as telas: abertura anual, panorama geral, ritmo do ano, destinos e rotas, eficiencia operacional, compliance e politica, economia e impacto, resumo com IA, wrapped do time e proximas acoes.
Inclua uma area onde o gestor possa abrir os wrappeds individuais dos colaboradores.
Use dados mockados plausiveis de viagens corporativas.
A interface deve parecer premium, narrativa e pronta para pitch.
```

## Prompt para Wrapped Onfly do colaborador

```text
Use $onfly-retrospective-design.
Implemente o Wrapped Onfly para o perfil colaborador.
Crie as telas: abertura pessoal, resumo do ano, mapa pessoal, ritmo de viagem, estilo de viajante, habitos, eficiencia pessoal, resumo com IA, dica para o proximo ciclo e fechamento.
Use uma linguagem mais pessoal, mas ainda corporativa.
A interface deve ser mobile-first e tambem funcionar bem no desktop.
```

## Prompt para gestor visualizando o wrapped do colaborador

```text
Use $onfly-retrospective-design.
Implemente a visao do gestor abrindo o Wrapped individual de um colaborador.
Crie uma experiencia curta com: entrada no perfil, resumo do colaborador, perfil de comportamento, pontos de atencao ou destaque e recomendacao gerencial com IA.
A interface deve deixar claro que e uma visao de acompanhamento e desenvolvimento, nao de fiscalizacao.
```

## Prompt para Capsula de Viagens do gestor

```text
Use $onfly-retrospective-design.
Implemente a Capsula de Viagens mensal para o perfil gestor.
Crie as telas: retrato do mes, tendencia de viagens e gastos, politica e excecoes, times ou pessoas para acompanhar, capsulas individuais do time e recomendacao de IA para o proximo mes.
Mantenha a experiencia mais objetiva do que o Wrapped, mas ainda visualmente forte.
```

## Prompt para Capsula de Viagens do colaborador

```text
Use $onfly-retrospective-design.
Implemente a Capsula de Viagens mensal para o perfil colaborador.
Crie as telas: seu mes em viagens, rotas e reservas, insight de habito, destaque de despesa ou compliance e sugestao de IA para o proximo mes.
A experiencia deve ser rapida de consumir e util para retorno recorrente ao produto.
```

## Prompt de implementacao enxuta

```text
Use $onfly-retrospective-design.
Implemente apenas a casca de interface do [produto] no perfil [perfil].
Use no maximo 5 telas.
Priorize estrutura visual, narrativa e componentes.
Use mocks estaticos.
Nao implemente backend.
```
